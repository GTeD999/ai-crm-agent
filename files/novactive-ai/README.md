# Novactive AI Manager — спецификация для Codex

AI-менеджер агентства недвижимости «Новактив». MVP: Telegram-бот, который квалифицирует лиды, отвечает на вопросы по объектам и передаёт «горячих» клиентов в Битрикс24.

## Стек

- **LLM**: OpenAI `gpt-4o` (диалог) + `gpt-4o-mini` (классификация/суммаризация) + `text-embedding-3-small` (RAG)
- **Backend**: Python 3.11+, FastAPI, aiogram 3.x (Telegram), httpx (async)
- **БД**: Supabase (Postgres 15 + pgvector + Row Level Security)
- **Очереди/планировщик**: APScheduler (для MVP), Celery+Redis (если нагрузка вырастет)
- **CRM**: Битрикс24 (через входящий webhook + REST API)
- **Хостинг**: любой VPS с Docker (Timeweb, Selectel, Yandex Cloud)

## Как использовать с Codex

1. Загрузите всю папку в Codex одним пакетом.
2. Дайте Codex задачу по одному модулю за раз — не пытайтесь сгенерировать всё одной командой.
3. Порядок реализации (от простого к сложному):
   1. Поднять Supabase, накатить `schemas/database.sql`
   2. Реализовать модели и репозитории (`code/project-structure.md` → `db/`)
   3. Реализовать OpenAI клиент с tools (`schemas/openai-tools.json`)
   4. Реализовать Telegram-хендлеры (aiogram)
   5. Интеграцию с Битрикс24
   6. Follow-up планировщик
   7. Тесты по `docs/testing.md`

## Что должен делать AI-менеджер

1. Принимать сообщения в Telegram
2. Запрашивать согласие на обработку ПД (152-ФЗ для РФ)
3. Квалифицировать лид: бюджет, тип, локация, кол-во комнат, сроки, цель (жильё/инвестиция), ипотека
4. Искать объекты в БД по критериям (семантический поиск + фильтры)
5. Отвечать на вопросы по конкретным объектам
6. Бронировать просмотры (запись слота)
7. Передавать «горячие» лиды менеджеру в Битрикс24
8. Делать follow-up через 1/3/7 дней

## Критичные правила

- **Никогда не выдумывать объекты, цены, метраж** — только данные из БД
- **Никогда не давать юридических гарантий** — направлять к юристу агентства
- **Согласие на ПД до сбора данных** — обязательно
- **Логировать всё** — для аудита и обучения
- **Hand-off менеджеру** должен включать саммари диалога
